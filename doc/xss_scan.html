2. Temuan A — Reflected XSS (GET)

Endpoint

xss_get.php — parameter firstname

Payload

http://localhost:8080/xss_get.php?firstname=<script>alert('XSS-bWAPP')</script>&lastname=test&form=submit

Bukti (response server)

Payload dipantulkan kembali ke HTML tanpa output encoding:

Welcome <script>alert('XSS-bWAPP')</script> test

Karena tag <script> tidak di-escape menjadi &lt;script&gt;, browser korban akan mengeksekusi JavaScript tersebut. Serangan dipicu lewat link berbahaya yang dikirim ke korban.

3. Temuan B — Stored XSS (Blog)

Endpoint

xss_stored_1.php — field entry (POST)

Payload

POST /xss_stored_1.php
entry=<script>alert('Stored-XSS-by-Audit')</script>&blog=submit&entry_add=

Bukti Persistence (request GET terpisah setelah penyimpanan)

Payload tersimpan permanen di database dan dirender mentah ke setiap pengunjung halaman blog:

<td><script>alert('Stored-XSS-by-Audit')</script></td>

Berbeda dari Reflected XSS, payload ini aktif bagi seluruh pengguna yang membuka halaman, tanpa perlu link khusus. Catatan: payload uji telah dihapus kembali setelah verifikasi untuk menjaga kebersihan lab.

5. Rekomendasi Remediasi

  1. Terapkan output encoding sesuai konteks (HTML entity encoding) pada semua data yang ditampilkan, mis. htmlspecialchars() di PHP.
  2. Lakukan input validation dengan whitelist pada sisi server.
  3. Pasang Content Security Policy (CSP) untuk membatasi eksekusi script.
  4. Set flag HttpOnly dan Secure pada cookie session agar tidak dapat diakses JavaScript.
  5. Gunakan framework/template engine yang melakukan auto-escaping secara default.
Dokumen ini dihasilkan untuk keperluan mata kuliah IT Audit. Pengujian dilakukan pada lingkungan lab terisolasi (localhost). bWAPP adalah aplikasi yang sengaja dibuat rentan untuk tujuan edukasi. Jangan gunakan teknik ini pada sistem tanpa izin.
