Catatan metodologi (penting untuk interpretasi): Skrip vulners mencocokkan kerentanan berdasarkan banner versi (CPE) saja, bukan dengan menguji eksploitasi secara langsung. Distribusi paket Ubuntu sering melakukan backport tambalan keamanan tanpa menaikkan nomor versi, sehingga sebagian CVE dalam daftar mungkin sudah ditambal pada build spesifik ini. Daftar ini valid sebagai indikator permukaan risiko komponen usang, dan untuk konfirmasi per-CVE diperlukan verifikasi versi paket atau pengujian eksploitasi terkontrol. Sebagian entri (mis. ber-ID tahun mendatang) juga dapat berupa noise pencocokan.

4. PHP 5.5.9

PHP 5.5.9 tidak dipindai vulners sebagai layanan terpisah (terdeteksi via header, bukan port khusus), namun statusnya yang End-of-Life sejak 2016 menjadikannya temuan A06 tersendiri. Cabang PHP 5.5 tidak lagi menerima pembaruan keamanan dan memiliki sejumlah besar CVE yang diketahui. Disarankan migrasi ke versi PHP yang masih didukung (8.x).

6. Rekomendasi Remediasi

  1. Perbarui Apache httpd ke rilis stabil terkini (cabang 2.4.x yang didukung) atau gunakan paket OS yang aktif menerima tambalan keamanan.
  2. Migrasi PHP dari 5.5.9 ke versi yang masih didukung (PHP 8.x).
  3. Tegakkan manajemen patch dan pemindaian komponen berkala (SCA) sebagai bagian dari siklus pemeliharaan.
  4. Sembunyikan/minimalkan banner versi (ServerTokens Prod, nonaktifkan X-Powered-By) untuk mengurangi kemudahan enumerasi oleh penyerang (mitigasi berlapis, bukan pengganti patch).
